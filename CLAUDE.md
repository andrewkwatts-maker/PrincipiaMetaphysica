### Validation Pattern

All simulations inherit from SimulationBase and implement:

```python
class MySimulation(SimulationBase):
    def execute(self, registry, verbose=False):
        # Check dependencies
        self.check_dependencies(registry)

        # Run computation
        results = self.compute()

        # Validate outputs
        self.validate_results(results)

        # Register in PMRegistry
        registry.register_parameter("param_id", value, metadata)

        return results
```

## Important Constraints

### Sterility Requirements

1. **No Magic Numbers**: All numeric literals must be documented in FormulasRegistry
2. **No Ghost Literals**: Use [identify_ghost_literals.py](simulations/core/identify_ghost_literals.py) to detect violations
3. **SSoT Compliance**: All values must trace back to Ten Pillar Seeds or experimental inputs
4. **Demon Lock**: Parameter values are immutable once registered

### Naming Conventions

- **Legacy names** (e.g., `alpha4`, `alpha5`) are deprecated
- Use **canonical names** (e.g., `theta_23_rad`, `delta_m21_sq`)
- Check [simulations/core/FormulasRegistry.py](simulations/core/FormulasRegistry.py) for approved naming
- Tests enforce strict naming compliance

### Precision Requirements

- Use `Decimal` from `decimal` module for high-precision calculations
- Context locked to 64 digits with ROUND_HALF_EVEN
- See `lock_geometric_context()` in FormulasRegistry

### Version Tracking

- Main version in [config.py](config.py): `VERSION = "23.3.2"`
- Simulations version in [simulations/__init__.py](simulations/__init__.py): `__version__ = "23.0"`
- Keep CHANGELOG.md updated for major changes
