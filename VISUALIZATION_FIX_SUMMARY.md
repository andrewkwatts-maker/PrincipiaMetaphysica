## Known Issues and Warnings

### Font Warnings (Non-Critical)
```
UserWarning: Glyph 8712 (\N{ELEMENT OF}) missing from font(s) Times New Roman.
```

**Impact**: Minimal. The ∈ (element of) symbol may not render perfectly in Times New Roman, but matplotlib falls back to DejaVu Serif. PDF output is unaffected.

**Fix (Optional)**: Install additional math fonts or use different serif font.

## Maintenance Notes

### Adding New Visualizations

To add a new visualization to the automatic generation:

1. Create visualization script in `simulations/visualizations/`
2. Ensure it has a callable entry point (e.g., `generate_my_plot()`)
3. Add import and call in `_generate_visualizations()` method
4. Test with `test_visualization_generation.py`

**Example**:
```python
# In _generate_visualizations() method:
try:
    if self.verbose:
        print("\n[VIZ] Generating My New Plot...")
    from simulations.visualizations.my_new_plot import generate_plot
    generate_plot()
    if self.verbose:
        print("  [OK] My new plot generated")
except Exception as e:
    visualization_errors.append(f"My new plot: {e}")
    if self.verbose:
        print(f"  [WARN] My new plot failed: {e}")
```

### Visualization Script Template

All visualization scripts should:
1. Output to `AutoGenerated/plots/`
2. Have a `main()` function for standalone execution
3. Have a callable entry point function (e.g., `generate_X()`)
4. Use pathlib for cross-platform compatibility
5. Handle matplotlib backend gracefully
6. Return the output path for logging
