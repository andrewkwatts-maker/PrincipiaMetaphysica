# Principia Metaphysica - Web Interface Guide

## Quick Start

**IMPORTANT**: The web interface MUST be served via HTTP server, not opened directly with `file://` protocol.

### Method 1: Using the Built-In Server (Recommended)

```bash
# Start the development server
python serve.py

# Server will automatically:
# - Find an available port (default: 8000)
# - Open your browser to http://localhost:8000
# - Serve all files with proper CORS headers
```

### Method 2: Using Python HTTP Server

```bash
# Start simple HTTP server
python -m http.server 8000

# Then open your browser to:
# http://localhost:8000/index.html
```

### Method 3: Custom Port

```bash
# Use a specific port
python serve.py --port 8080

# Or
python serve.py 8080
```

---

## Why File Protocol Doesn't Work

When you open HTML files directly (`file:///path/to/index.html`), browsers enforce strict **CORS (Cross-Origin Resource Sharing)** restrictions that prevent JavaScript from:

- Loading JSON files from `AutoGenerated/` folder
- Fetching data with `fetch()` or `XMLHttpRequest`
- Accessing local resources dynamically

**Symptoms of file:// protocol issues**:
- Pages load but show "Loading..." forever
- Browser console shows CORS errors
- JSON data files fail to load
- Simulations page shows empty categories

**Solution**: Always use an HTTP server (methods above).

## Troubleshooting

### Problem: Simulations page shows empty categories

**Cause**: `AutoGenerated/simulations-index.json` is empty or missing

**Fix**:
```bash
python simulations/Constants/generate_simulations_index.py
```

---

### Problem: Pages show "Loading..." forever

**Cause 1**: Opened with `file://` protocol instead of HTTP server

**Fix**: Use `python serve.py` instead of double-clicking HTML files

**Cause 2**: JSON files missing from AutoGenerated/ folder

**Fix**: Run full simulation pipeline
```bash
python simulations/run_all_simulations.py
```

---

### Problem: "Network Error" or "Failed to fetch"

**Cause**: CORS restrictions when using file:// protocol

**Fix**: Always use HTTP server (see Quick Start above)

---

### Problem: Port 8000 already in use

**Fix**: Use a different port
```bash
python serve.py --port 8001
```

---

### Problem: JavaScript errors in browser console

**Symptoms**:
- `Uncaught SyntaxError: Unexpected token '<'` (trying to parse HTML as JSON)
- `CORS policy: No 'Access-Control-Allow-Origin' header`
- `Failed to load resource: net::ERR_FILE_NOT_FOUND`

**Fix**: Ensure you're using HTTP server and all AutoGenerated files exist

## Offline Use

The web interface is designed for **full offline use**:

1. Clone repository: `git clone https://github.com/andrewkwatts-maker/PrincipiaMetaphysica.git`
2. Run simulations: `python simulations/run_all_simulations.py`
3. Start server: `python serve.py`
4. All data is local (no external API calls except MathJax CDN)

**Optional**: For fully offline use, download MathJax locally:
```bash
# Download MathJax
npm install mathjax@3

# Update HTML to point to local copy instead of CDN
```

## Production Deployment

For deploying to a web server (GitHub Pages, Netlify, etc.):

1. **Ensure all AutoGenerated files are up to date**:
   ```bash
   python simulations/run_all_simulations.py
   python simulations/Constants/generate_simulations_index.py
   ```

2. **Commit all files**:
   ```bash
   git add AutoGenerated/*.json
   git commit -m "Update generated data files"
   git push
   ```

3. **Deploy** (example for GitHub Pages):
   ```bash
   # GitHub Pages serves from root or docs/ folder
   # Ensure index.html is in root
   # Push to gh-pages branch or enable Pages in Settings
   ```

4. **Verify deployment**:
   - Visit live URL
   - Check browser console for errors
   - Test all pages load correctly
   - Verify simulations page shows 248 scripts
