## What is P-Value?

The p-value represents the probability of obtaining a chi-squared value **this low or lower** by chance, given the degrees of freedom:

```
p-value = 1 - chi2.cdf(chi_squared, dof)
        = 1 - chi2.cdf(5.751, 25)
        ≈ 0.9999774
```

### Interpretation by P-Value Range:

| P-Value Range | Interpretation | Status |
|--------------|----------------|--------|
| **p > 0.99** | Fit is "too good" - uncertainties likely overestimated | **CURRENT** |
| 0.95 < p < 0.99 | Very good fit - verify error estimates | Marginal |
| 0.05 < p < 0.95 | Appropriate fit quality | Target |
| 0.01 < p < 0.05 | Model slightly under-fits data | Marginal |
| **p < 0.01** | Model does not fit data | Failed |

**Conclusion**: p ≈ 1.0 means the fit is suspiciously good.

## Solution: Add Theory Uncertainties

### Prescription

For each **GEOMETRIC** or **DERIVED** parameter, add theory uncertainty in quadrature:

```
σ_total² = σ_experimental² + σ_theory²
```

Where:
```
σ_theory = max(
    0.01 × |predicted_value|,   # 1% baseline theory uncertainty
    α_leak × |predicted_value|  # Dark sector leakage ~ 0.408
)
```

### Example: w₀ with Theory Uncertainty

```
w₀_pred = -0.9583
w₀_exp = -0.957 ± 0.067 (experimental only)

σ_theory = 0.01 × 0.9583 = 0.00958  (~1% theory error)
σ_total = √(0.067² + 0.00958²) = √(0.00449 + 0.0000918) = 0.0670

New sigma: (0.9583 - 0.957) / 0.0670 = 0.0194σ
```

**Effect**: Almost no change because experimental error dominates.
