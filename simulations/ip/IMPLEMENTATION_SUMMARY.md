## Security Properties

### Cryptographic Strength

- **Algorithm**: SHA-256 (NIST FIPS 180-4)
- **Hash Space**: 2²⁵⁶ ≈ 10⁷⁷ possible values
- **Collision Resistance**: Computationally infeasible to find two inputs with same hash
- **Preimage Resistance**: Cannot reverse-engineer input from hash
- **Second Preimage Resistance**: Cannot find different input producing same hash

### Tamper Detection

Any modification to a prediction will:
1. Change the SHA-256 hash completely (avalanche effect)
2. Be detected by `verify_hashes.py`
3. Break the hash chain
4. Invalidate the timestamp

### Git Integration

- Each prediction linked to specific git commit
- Git commit timestamp provides temporal ordering
- Commit hash provides additional verification layer
- Full audit trail in git log

### Manual Verification (Python)

```python
import json
import hashlib

# Load registry
with open('AutoGenerated/discovery_hashes.json', encoding='utf-8') as f:
    data = json.load(f)

# Verify first prediction
entry = data['discoveries'][0]
prediction = entry['prediction']

# Compute hash
canonical = json.dumps(prediction, sort_keys=True, separators=(',', ':'))
computed = hashlib.sha256(canonical.encode('utf-8')).hexdigest()

# Check
assert computed == entry['discovery_hash'], "Hash mismatch!"
print(f"Verified: {prediction['name']}")
```
