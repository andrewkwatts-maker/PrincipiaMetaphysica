## How Hash Verification Works

Each prediction is hashed using SHA-256:

```python
import json, hashlib

# Example: Verify first prediction
prediction = {
    "name": "Dark Energy EOS Parameter w₀",
    "parameter_key": "cosmology.w0_derived",
    "value": -0.8461538461538461,
    "units": "dimensionless",
    "uncertainty": null,
    "prediction_type": "cosmological",
    "significance": "Derives w₀ = -11/13..."
}

# Compute hash
canonical = json.dumps(prediction, sort_keys=True, separators=(',', ':'))
hash_value = hashlib.sha256(canonical.encode('utf-8')).hexdigest()

# Result: 918f299f7874ef2021f47a3f99f6a74833bbe87840f9fd9bcb5cddaecd48d0ef
```

The hash proves:
- This exact prediction existed at this time
- The value hasn't been changed
- The git commit is authentic

## Security Properties

- **SHA-256**: 2²⁵⁶ possible hashes (cryptographically secure)
- **Collision Resistance**: Impossible to create two different predictions with same hash
- **Preimage Resistance**: Cannot reverse-engineer prediction from hash
- **Tamper Detection**: Any change to prediction completely changes hash

## Troubleshooting

### "Registry not found"

Run the generator first:
```bash
python simulations/ip/discovery_hash_registry.py
```

### "Hash mismatch"

This means the registry has been tampered with. Regenerate:
```bash
python simulations/ip/discovery_hash_registry.py
python simulations/ip/verify_hashes.py
```

### Unicode errors in console

This is cosmetic (Windows console encoding). The JSON file is correct (UTF-8).
